# i18n Phase 3 系統性修改清單

> 最後更新：2026-04-18
> 目標：UroSense 支援 5 國語言（zh-TW / en-US / ja-JP / ko-KR / vi-VN），解決「切英文後病患端頁面仍為中文」的回報 bug，並完成全站 i18n 漏洞掃除。
> 完成一項就把 `[ ]` 改成 `[x]`，並在後面加上完成日期與 commit hash。

---

## 原始問題與脈絡

- 使用者回報：切換英文後，「選擇症狀」與「填寫病史資料」頁仍顯示中文。
- 根本原因：
  1. 前端 intake 流程兩頁完全硬編碼中文，沒走 `useTranslation`。
  2. 後端 `chief_complaints` 只有 zh-TW 欄位，沒有多語 schema。
  3. API 無語言 resolver，也無中介層讀取 `Accept-Language`。
  4. `SupportedLanguage` 僅 zh-TW / en，沒有 ja-JP / ko-KR / vi-VN。
- 修復策略：expand-migrate-contract，不破壞 legacy 欄位；前端 namespace 拆分；beta 語言走 fallbackLng chain。
